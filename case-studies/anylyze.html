Project Overview

Anylyze represented a greenfield opportunity to revolutionize how sales teams interact with data. As the sole product designer, I was responsible for creating an entire no-code mobile platform from conception to market launch while supporting a development team of 25 engineers.

The challenge was multifaceted: design a platform that could handle complex data processing workflows while remaining intuitive enough for non-technical sales professionals to use in fast-paced field environments. This required not just interface design, but fundamental UX research into sales workflows and mobile interaction patterns for data-heavy applications.

Beyond the core product, I simultaneously created the Anylyze Design System from scratch, contributed to the Optymyze Design System, and developed a generic design framework that could support four different products across mobile, web, and desktop platforms.

The Challenge

Sales optimization tools typically required technical expertise and desktop environments, creating barriers for field sales teams. Key challenges included:

  • Complex data processing workflows needed simplification
  • Mobile-first design for field sales environments
  • No existing design patterns for mobile no-code platforms
  • Supporting 25 developers with consistent design direction
  • Creating scalable design systems across multiple products

The Solution

I developed a comprehensive design strategy focusing on mobile-native interactions and progressive disclosure:

  • Touch-first interface design for complex data workflows
  • Progressive disclosure to manage feature complexity
  • Comprehensive design system for rapid development
  • User research-driven interaction patterns
  • Cross-platform design system architecture

Design Process

1

Sales Workflow Discovery

Conducted extensive field research with sales teams to understand their daily workflows, pain points, and mobile usage patterns. Used interviews and contextual inquiries to map the sales optimization journey.

2

Mobile-First Architecture

Designed information architecture and interaction patterns specifically optimized for mobile devices, considering touch interfaces, limited screen real estate, and offline capabilities.

3

No-Code Interface Design

Created intuitive drag-and-drop interfaces and visual workflow builders that allowed non-technical users to create complex data processing pipelines through touch interactions.

4

Design System Creation

Built comprehensive design system from scratch, including mobile-specific components, interaction patterns, and documentation that enabled 25 developers to work consistently across the platform.

5

Cross-Platform Strategy

Extended design system architecture to support web and desktop versions while maintaining design consistency and enabling rapid scaling across Optymyze's product suite.

Key Learnings

Mobile No-Code Requires Fundamental Rethinking: Traditional no-code interfaces don't translate directly to mobile. Touch-first design requires new interaction paradigms and information architecture approaches.

Design Systems Enable Team Velocity: Creating comprehensive design systems upfront proved crucial for supporting 25 developers efficiently and maintaining consistency across rapid development cycles.

Field Research Is Critical for B2B Mobile: Understanding real-world sales environments and constraints was essential for creating truly usable mobile interfaces for professional use cases.
